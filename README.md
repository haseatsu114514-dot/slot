# 打つべきかカレンダー

Google スプレッドシートの収支データをもとに、六十干支ごとの「打つべきか / 打たないべきか」をカレンダーで見るための静的サイトです。

## この版で入っているもの

- 2026-04-07 を `辛亥` として固定した六十干支カレンダー
- 表示範囲は今日を基準に自動追従（既定: 過去 12 か月〜未来 3 か月）。月が変わっても更新作業は不要
- グリッド表示とリスト（アジェンダ）表示の切替。スマホはリスト表示が見やすい
- カレンダー上の左右スワイプで前月 / 翌月へ移動（スマホ）
- 日付ごとの評価、通変星、実績平均、占断予想の表示
- 成績入力フォームとセル長押しのクイック入力（± ボタンで負け額の符号を反転できる）
- 入力済み収支の編集・削除
- 同じ日付に同じ収支を入れたときの重複警告
- ローカル保存
- Google Apps Script 経由で Google Sheets に追加・更新・削除する同期構成

## ファイル構成

- `index.html`
  そのままブラウザで開けるフロント本体です。
- `kanshi-data.js`
  干支データ、干支計算、暫定スコア再計算ロジックです。
- `app.js`
  画面描画、フォーム処理、Sheets 同期処理です。
- `google-apps-script/Code.gs`
  シート連携用の Web アプリです。
- `google-apps-script/appsscript.json`
  Apps Script の設定です。

## ローカルで見る

ビルドは不要です。`slot/index.html` をそのまま開けます。

## テスト

```
node tests/kanshi-data.test.mjs
node tests/code-gs-parity.test.mjs
```

2 つ目は `Code.gs` と `kanshi-data.js` のスコアリングロジックが乖離していないかを照合します。
どちらかのスコア計算を変えたら必ず両方実行してください。

## Google Sheets とつなぐ手順

1. Google Apps Script に `google-apps-script/Code.gs` と `google-apps-script/appsscript.json` を貼り付けます。
2. 必要なら Script Properties に次を設定します。
   - `SPREADSHEET_ID`
   - `API_SECRET`
   - `MASTER_SHEET_NAME`
   - `RESULTS_SHEET_NAME`
3. Apps Script 側で `setupSheets()` を一度実行します。
4. 続けて `seedMasterSheet()` を一度実行します。
5. Web アプリとしてデプロイします。
6. `index.html` 内の `window.SLOT_APP_CONFIG.syncEndpoint` に Web アプリ URL を入れます。
7. `API_SECRET` を使う場合は `window.SLOT_APP_CONFIG.syncSecret` に同じ文字列を入れます。

### Apps Script を更新する手順 (再デプロイ)

リポジトリの `Code.gs` を変えたら、シート側にも反映が必要です。難しくありません:

1. Google スプレッドシートを開く → 上のメニュー「拡張機能」→「Apps Script」
2. 左のファイル一覧で `コード.gs` (Code.gs) をクリック → 中身を全部選択して削除 →
   リポジトリの `google-apps-script/Code.gs` の中身を全部コピーして貼り付け → Ctrl+S (Macは⌘S) で保存
3. 右上の青い「デプロイ」→「デプロイを管理」→ 鉛筆マーク (編集) →
   「バージョン」を「**新バージョン**」に変える →「デプロイ」
4. おわり。URL は変わらないので他の設定はそのままで OK

※ 手順 3 の「新バージョン」を忘れると、保存しても公開側に反映されません (一番多いミス)。

### 過去実績のシート移行

焼き込みの過去実績 122 日分を「実績入力」シートへ移すスクリプトがあります。
移行するとシート上で全履歴を一括管理 (閲覧・修正・削除) できます。

```
node scripts/migrate-seed-history.mjs        # ドライラン (確認のみ)
node scripts/migrate-seed-history.mjs --run  # 実際に書き込む
```

- 各行のメモに `[過去実績移行]` の目印が入り、アプリはこれを見て焼き込みデータの
  使用をやめ、シートのエントリ全量から実績を再構成します (二重計上しない)
- **公開サイトが移行対応版になってから実行すること**。旧版アプリのまま移行すると
  収支が二重に表示されます
- **デプロイ済み Apps Script が現行版 (deleteResult 対応) であること**。スクリプトは
  起動時に確認し、旧版なら中止します。旧版の `addResult` は成功しても `appended` を
  返さないため、成功を失敗と誤判定して 3 重書き込みする事故が起きます
- 二重移行はスクリプト側で防止されます (目印つきの行があれば何もしない)

> **注意 (推定日付について)**
> 焼き込み 122 日分は本当の日付を持たず、干支から逆算した推定日付になります。
> 移行すると「実際には打っていない日に大きな収支」が必ず生じます。日付別・曜日別の
> 検証をしたい場合のみ移行し、通常は焼き込み (コード内の集計値) のままで構いません。
> 焼き込みのまま運用しても、アプリは「焼き込み 122 日分 + シートの実記録」を合算して
> 表示するので、両方のデータは生きています。

### 移行を取り消す (ロールバック)

移行で増えた `[過去実績移行]` 行をすべて消し、移行前 (実記録のみ) に戻します。
焼き込み 122 日分は `kanshi-data.js` 内にコードとして残るので消えません。

```
node scripts/rollback-seed-migration.mjs        # ドライラン (件数確認のみ)
node scripts/rollback-seed-migration.mjs --run  # 実際に削除する
```

- 削除は行の id 指定で、メモに目印の無い実記録には一切触れません
- **このスクリプトは現行版 Apps Script (deleteResult 対応) が必要です**。旧版のままなら
  下の「シート上で手動削除」を使ってください

#### シート上で手動削除 (再デプロイ不要)

API が使えない (旧版デプロイのまま) 場合は、スプレッドシートで直接消せます:

1. スプレッドシートの「実績入力」シートを開く
2. メニュー「データ」→「フィルタを作成」
3. **H 列 (メモ)** のフィルタ▽ →「値でフィルタ」で `[過去実績移行]` を含む行だけ表示
4. 表示された行をすべて選択 (行番号をドラッグ) → 右クリック →「選択した行を削除」
5. フィルタを解除 (「データ」→「フィルタを削除」)

メモが空の実記録はフィルタで隠れるので、誤って消す心配はありません。

### セキュリティ上の注意

`index.html` はそのまま公開されるため、`syncSecret` もソースから誰でも読めます。
GitHub Pages などで公開する場合、このシークレットは「URL を知らない人の誤操作を防ぐ」程度の意味しかありません。
書き込まれて困るデータを扱うなら、Apps Script のデプロイを「自分のみ」アクセスに絞るか、定期的にシークレットと Web アプリ URL を変更してください。

### 既存データ修正

この版の Apps Script には `repairRequestedEntries` アクションが入っています。
Web アプリを再デプロイしたあとにこのアクションを実行すると、2026-05-20 の `7000` を `5000` に更新し、2026-05-14 の `-11000` 重複4件を1件に整理します。

## スコア反映について

元データの `score` を初期値として持ちつつ、追加された実績で平均収支が変わったぶんだけスコアを補正します。
実績日数が増えるほど占断より実績側の重みが自然に増える「シュリンクブレンド」方式です（詳細は `CLAUDE.md` のスコアリング仕様）。

元シート側に独自の数式ロジックがある場合は、Apps Script 側の `computeLiveScore_` をその式に合わせて置き換えると、フロントとシートの挙動をそろえられます。

## 暦データの保守

- 月干支の節入時刻（`kanshi-data.js` の `MONTH_PILLAR_TRANSITIONS`）は **2028 年末まで** 収録済みです。
  毎年 2 月に国立天文台の暦要項が公表されたら、翌年分を 1 年ぶん追記してください。
  データが切れそうになると `node tests/kanshi-data.test.mjs` が失敗して知らせます。
- 日家九星の切替日（`KYUSEI_SWITCH_POINTS`）も 2028 年分まで収録済みです（180 日周期の甲子日）。
